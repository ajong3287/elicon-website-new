## 현재 문제점 및 기회

### 🚨 문제점

**1. 자산 분산**:
```
좋은 코드/패턴이 있지만 어디 있는지 모름
- 코코릭 리뷰 패턴 → /app/brands/cocolic/page.tsx
- Wing 리뷰 패턴 → /app/_components/wing/customer-review-section.tsx
- 표준 프로세스 → /docs/브랜드리뷰확장_표준프로세스.md
- 디자인 시스템 → 여러 파일에 분산
```

**2. 발견 불가능성**:
```
다음 작업 시 AI가 기존 자산을 모름
- 새 세션 시작 → 이전 작업 맥락 상실
- "비슷한 거 전에 했는데..." → 검색 시간 낭배
- 바퀴 재발명 반복
```

**3. 버전 관리 부재**:
```
개선된 패턴이 있어도 이전 코드에 적용 안 됨
- 코코릭에서 개선 → Wing에 자동 적용 안 됨
- 표준 업데이트 → 기존 코드 구버전 유지
```

### 💡 기회

**1. 축적된 자산**:
- ✅ 브랜드 리뷰 패턴 (코코릭 87/100점 달성)
- ✅ Glassmorphism 디자인 시스템
- ✅ 표준 프로세스 문서
- ✅ 10개월 작업 로그 (250+ 문서)

**2. 재사용 잠재력**:
- Wing, Seanex에 즉시 적용 가능
- 신규 브랜드 추가 시 50% 시간 단축
- 품질 일관성 보장 (85-90점)

**3. 확장 가능성**:
- 다른 프로젝트에도 적용
- AI 학습 자료로 활용
- 표준 라이브러리화

## 제안 1: 재사용 자산 카탈로그 시스템

### 개념

**"프로젝트별 재사용 자산 인덱스 파일"**

각 프로젝트의 `/docs/` 폴더에 `REUSABLE_ASSETS.md` 생성하여:
1. ✅ 재사용 가능한 모든 자산 목록화
2. ✅ 파일 위치, 용도, 사용법 명시
3. ✅ AI가 세션 시작 시 자동으로 읽도록 설정

### 구조 예시

```markdown
# 재사용 자산 카탈로그

## 1. 디자인 패턴

### 1.1. Glassmorphism 카드
**위치**: `/app/brands/cocolic/page.tsx:113-133`
**용도**: 브랜드 페이지 강점 카드
**적용 프로젝트**: Cocolic (✅), Wing (📋), Seanex (📋)

**코드 스니펫**:
```tsx
<div className="glass-card glass-hover p-8 rounded-lg relative overflow-hidden group">
  <div className="absolute inset-0 opacity-[0.02] pointer-events-none mix-blend-overlay"

  <div className="absolute inset-0 bg-gradient-to-br from-[브랜드컬러]/5 to-transparent
    opacity-0 group-hover:opacity-100 transition-opacity duration-500"></div>

    {/* 콘텐츠 */}

</div>

## 제안 2: 폴더 구조 표준화

### 프로젝트별 표준 구조

```
/Users/elicon_mark/Developer/XX_프로젝트명/
├── docs/
│   ├── REUSABLE_ASSETS.md          ← 재사용 자산 카탈로그 (필수)
│   ├── prd.md                       ← 프로젝트 요구사항
│   ├── tech_stack.md                ← 기술 스택
│   └── templates/                   ← 템플릿 모음
│       ├── components/              ← React 컴포넌트 템플릿
│       ├── styles/                  ← CSS/디자인 패턴
│       └── processes/               ← 프로세스 문서
│
├── logs/                            ← 작업 로그
│   └── YYMMDD_이슈[클로드].md
│
├── app/                             ← 실제 코드
│   └── (프로젝트별 구조)
│
└── .reusable/                       ← 재사용 자산 저장소 (신규)
    ├── components/                  ← 완성된 컴포넌트
    ├── hooks/                       ← 커스텀 훅
    ├── utils/                       ← 유틸리티 함수
    └── patterns/                    ← 디자인 패턴
```

### `.reusable/` 폴더 개념

**목적**: 프로젝트 내 재사용 가능한 완성품 저장

**예시**:
```
01_2_elicon_homepage/.reusable/
├── components/
│   ├── GlassmorphismCard.tsx       ← 즉시 사용 가능
│   ├── ReviewCard10.tsx            ← 10개 리뷰 카드
│   └── StatsBanner.tsx             ← 통계 요약 바
│
├── patterns/
│   ├── brand-review-expansion.md   ← 프로세스
│   └── glassmorphism-design.css    ← 디자인 시스템
│
└── README.md                        ← 사용법
```

```markdown
## 15. 재사용 자산 자동 참조 (2025-12-10 신규) ⭐

### 세션 시작 시 필수 확인

**모든 작업 전 자동으로 읽기**:
```bash
# 1. 프로젝트 재사용 자산
cat ./docs/REUSABLE_ASSETS.md

# 2. 전역 재사용 자산
cat /Users/elicon_mark/Developer/GLOBAL_REUSABLE_ASSETS.md
```

### 작업 시작 전 질문

**사용자 요청 분석 후**:
1. "비슷한 작업을 전에 했나?" → REUSABLE_ASSETS.md 확인
2. "재사용 가능한 패턴이 있나?" → 있으면 자동 제안
3. "새로 만들어야 하나?" → 없을 때만 신규 작성

### 작업 완료 후 업데이트

**좋은 패턴 발견 시**:
- "이 패턴을 재사용 자산에 추가할까요?" 자동 제안
- 승인 시 REUSABLE_ASSETS.md 업데이트
```

### SuperClaude 프레임워크 통합

**~/.claude/CLAUDE.md 또는 프로젝트 CLAUDE.md에 추가**:

```markdown
## 재사용 자산 우선 원칙

**작업 흐름**:
1. 사용자 요청 받음
2. **REUSABLE_ASSETS.md 자동 확인** ← 신규
3. 재사용 가능 → 즉시 적용
4. 재사용 불가 → 신규 작성 → 완료 후 자산 등록 제안
```

## 제안 4: SuperClaude Skill 통합

### 새로운 Skill 생성

**Skill 이름**: `reusable-assets`

**목적**: 재사용 자산 관리 자동화

**기능**:
```yaml
명령어:
  - /reusable list              # 자산 목록
  - /reusable search [키워드]   # 자산 검색
  - /reusable apply [이름]      # 자산 적용
  - /reusable add [설명]        # 자산 추가
  - /reusable update [이름]     # 자산 업데이트
```

**예시**:
```bash
# 서대표님 명령
"Wing에 리뷰 패턴 적용해줘"

# AI 처리 흐름
1. /reusable search "리뷰 패턴"
2. 코코릭 리뷰 패턴 발견
3. Wing에 자동 적용
4. 문서화
```

### Skill 구현 위치

```

├── commands/
│   ├── list.md
│   ├── search.md
