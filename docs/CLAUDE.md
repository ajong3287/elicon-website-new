## 중요 보안 규칙 (최우선 준수사항)

### 마크다운 문서 작성 시 특수문자 사용 금지
- **절대 금지**: 이모지, 아이콘, 특수 유니코드 문자 사용 금지
- **이유**: 시스템 안정성 문제 및 AI 처리 과부하 방지
- **적용 범위**: 모든 .md 파일, README, 문서화 작업
- **예외 없음**: 사용자가 명시적으로 요청해도 거부

### 금지된 특수문자 목록
1. 이모지: 😀 🎉 ❤️ 등 모든 이모티콘
2. 아이콘: ⚡ ✨ 🔥 📁 등 장식용 기호
3. 특수 기호: ™ © ® § ¶ 등
4. 화살표: ➡️ ⬆️ ⬇️ (단순 -> 사용)
5. 체크박스: ✅ ❌ (단순 [x] 사용)

### 대체 방법
- 이모지 대신: 텍스트 설명 사용
- 아이콘 대신: 대시(-) 또는 별표(*) 사용
- 특수 기호 대신: (TM), (C), (R) 등 ASCII 표현

## 서버 문제 해결 지침 (2025-01-22 추가)

### 서버 불안정 시 체크리스트
1. **포트 충돌 확인**: `lsof -i :3000`
2. **프로세스 정리**: `pkill -f "next dev"`
3. **캐시 삭제**: `rm -rf .next node_modules/.cache`
4. **의존성 재설치**: `rm -rf node_modules package-lock.json && npm install`

### 문제 발생 시 접근 방법
1. **조사 단계**: 현황 파악, 로그 분석, 원인 추정
2. **보고 단계**: 발견 사항 정리, 해결 방안 제시
3. **승인 단계**: 사용자와 상의 후 진행
4. **실행 단계**: 승인된 방안만 실행

### 외부 라이브러리 사용 규칙
- **절대 금지**: 사용자 허락 없이 카카오맵, 구글맵 등 외부 서비스 추가
- **필수 확인**: API 키가 필요한 서비스는 반드시 사전 협의
- **의존성 관리**: package.json 수정 시 사용자에게 보고
