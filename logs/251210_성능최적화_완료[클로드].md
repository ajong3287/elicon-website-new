### 2. Next.js 이미지 최적화 설정 (기존 완벽 설정 확인)

#### AVIF/WebP 자동 변환
```typescript
formats: ['image/avif', 'image/webp']
```
- ✅ AVIF: WebP보다 20-30% 더 작은 파일 크기
- ✅ WebP: JPEG 대비 25-35% 작은 파일 크기
- ✅ 자동 폴백: 브라우저 지원에 따라 자동 선택

#### 반응형 이미지 크기
```typescript
deviceSizes: [640, 750, 828, 1080, 1200, 1920, 2048, 3840]
imageSizes: [16, 32, 48, 64, 96, 128, 256, 384]
```
- ✅ 디바이스에 최적화된 크기 자동 제공
- ✅ 불필요한 데이터 전송 방지
- ✅ srcset 자동 생성

#### 이미지 품질 설정
```typescript
qualities: [75, 90]
```
- ✅ 75: 기본 품질 (파일 크기 균형)
- ✅ 90: 고품질 (필요 시)

#### 외부 이미지 지원
- ✅ Instagram CDN 허용 (`scontent.cdninstagram.com`)
- ✅ 보안 패턴 적용 (HTTPS only)

### 3. 폰트 최적화 (기존 완벽 설정 확인)

#### Google Fonts 최적화
**사용 폰트**:
- Playfair Display (헤딩, 브랜드)
- Inter (본문)
- JetBrains Mono (코드)

**Next.js Font 최적화**:
```typescript
const playfair = Playfair_Display({
  variable: "--font-playfair",
  subsets: ["latin"],
  display: "swap",
});
```

**적용된 최적화**:
- ✅ `next/font/google` 사용 (자동 최적화)
- ✅ `display: "swap"` (FOUT 방지)
- ✅ CSS 변수 (`--font-playfair`)
- ✅ Latin subset만 로드 (불필요한 문자 제외)

**폰트 로딩 전략**:
- ✅ Critical CSS에 폰트 포함
- ✅ 자체 호스팅 (Google Fonts 자동 다운로드)
- ✅ 사전 로드 (preload)
- ✅ 폰트 파일 압축 (woff2)

### 4. PWA 캐싱 최적화 (기존 완벽 설정 확인)

#### 폰트 캐싱 전략
```javascript
// Google Fonts Webfonts - 1년 캐싱
handler: 'CacheFirst'
maxAgeSeconds: 365 * 24 * 60 * 60

// Google Fonts Stylesheets - 1주일 캐싱
handler: 'StaleWhileRevalidate'
maxAgeSeconds: 7 * 24 * 60 * 60
```

#### 이미지 캐싱 전략
```javascript
// 정적 이미지 - 1일 캐싱
handler: 'StaleWhileRevalidate'
maxEntries: 64
maxAgeSeconds: 24 * 60 * 60

// Next.js 이미지 최적화 - 1일 캐싱
handler: 'StaleWhileRevalidate'
cacheName: 'next-image'
```

#### 기타 자산 캐싱
- ✅ JavaScript: StaleWhileRevalidate (1일)
- ✅ CSS: StaleWhileRevalidate (1일)
- ✅ 오디오/비디오: CacheFirst (1일)
- ✅ JSON/XML: NetworkFirst (1일)

**캐싱 전략 요약**:
- **CacheFirst**: 네트워크 요청 최소화 (폰트, 오디오, 비디오)
- **StaleWhileRevalidate**: 빠른 응답 + 백그라운드 갱신 (이미지, CSS, JS)
- **NetworkFirst**: 최신 데이터 우선 (API, JSON)
