## 💡 정리 원칙

### 1. 파일 역할 명확화

- **docs/**: 프로젝트 이해를 위한 영구 문서
- **logs/**: 작업 히스토리 추적 (날짜 순)
- **scripts/**: 개발/테스트 자동화
- **reports/**: 성능/품질 측정 결과

### 2. 명명 규칙 준수

- 로그 파일: `YYMMDD_주제[작성자].md`
- 문서 파일: `의미있는_이름.md`
- 스크립트: `동사-명사.sh`

### 3. Git 최적화

- 자동 생성 파일 → `.gitignore`
- 바이너리 파일 → `archived_files/` 또는 외부 저장
- 소스 코드만 Git 추적
